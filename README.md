# Claude Session Manager

[Claude Code](https://claude.com/claude-code) の会話履歴・メモリ・エージェントを管理する VS Code 拡張機能。

ブックマーク、タグ付け、メモ、Markdownプレビュー、エージェント管理、組織図など、Claude Code本体にない会話・エージェント管理機能を提供します。

---

## 主な機能

### 会話管理

- **会話一覧** — 全プロジェクトの会話を日付別（今日/昨日/今週/今月/それ以前）に自動分類
- **モデル表示** — モデル頭文字（Ｏ=Opus / Ｓ=Sonnet / Ｈ=Haiku）とメッセージ件数を表示
- **プレビュー** — チャット形式で会話内容を表示。Markdownレンダリング対応（コードブロック・テーブル・見出し・リスト）
- **リンクのクリック対応** — プレビュー内のURLはブラウザ、Windowsファイルパスはエディタで開く（行番号ジャンプ付き）
- **ブックマーク** — 大事な会話をお気に入り登録。専用セクションに一覧表示
- **タグ** — 自由なカテゴリで分類。タグ別グループ表示
- **メモ** — 各会話に役割や目的のメモを記入可能。500ms遅延で自動保存
- **リネーム** — 会話にわかりやすい名前を設定。Claude Code側と双方向同期
- **検索** — タイトル・プロジェクト・ブランチ名でフィルタ
- **Claude Codeで開く** — 右クリックからVS CodeのClaude Codeパネルで会話を再開（Antigravity等フォーク対応）
- **セッションIDコピー** — 右クリックからセッションIDをクリップボードにコピー
- **セッションパスをコピー** — 右クリックからJSONLファイルのフルパスをコピー（他エージェントに読ませる際に便利）

### ライブセッション検出

- Claude Codeで現在使用中のセッションをリアルタイム検出
- 使用中の会話は緑の **●** アイコンで表示
- PID生存チェック＋5秒ポーリングでゾンビセッションも自動クリーンアップ

### 子エージェントのツリー表示

- セッション配下の子エージェント（`subagents/` フォルダ内の JSONL）を自動検出
- 親セッションを展開すると子エージェントをツリー表示
- タイプ別アイコン: 🔍探索 / 📋計画 / 🔧汎用 / 📖ガイド

### エージェント管理

- **エージェント登録** — セッションを右クリック → 「エージェントとして登録」
  - Webviewフォームで部署名・役割・モデル・セッション運用（固定/使い捨て）・上司・作業フォルダ・ルールファイルを一括入力
  - カード型ラジオボタンでモデル・運用モードを選択、フォルダ/ファイル選択ダイアログ連携
  - ルールファイル未設定時はひな形を自動生成
- **エージェントプレビュー** — エージェント一覧でクリックすると読み取り専用プレビューを表示
  - 名前・モデル・上司・役割・子エージェント一覧・ルールファイル内容を確認
  - 「設定」ボタンで編集フォームに切り替え
  - セッション名リンクからセッション履歴プレビューへ遷移
- **エージェント管理サイドバー** — 登録済みエージェントの一覧表示
  - ライブ状態（🟢/⚪）・モデルバッジ・ルールファイル行数/サイズを表示
  - 使い捨てエージェントは「使い捨て」ラベル表示
  - 右クリックからセッション紐づけ・設定編集・ルールファイル編集・削除
  - 「取締役」はツリーの最上位に表示
- **セッションを開く** — 紐づけ済みエージェントのセッションをClaude Codeで開く
- **セッションを新しくする** — 引き継ぎメッセージ（遺言）を残して新セッションに切り替え
- **設定編集** — 登録済みエージェントの設定を既存値表示のまま編集
- **ルールファイル編集** — 右クリックから直接エディタで開く
- **プレビューヘッダのバッジ** — 登録済みエージェントのセッションにモデル・名前・役割を表示。「設定編集」「ルールファイル」リンク付き
- **セッション一覧のエージェント名** — 登録済みセッションの description に `🤖部署名` を表示

### 組織図・ステータスバー

- ステータスバー（左下）に「🟢 2 👥 5」形式で表示（🟢=動作中数、👥=登録総数）
- ホバーで動作中エージェント名リストをツールチップ表示
- クリックすると Webview パネルで組織図を表示
- 各カードにモデルバッジ・役割・セッション状態を表示
- **🕐 ボタン** でカードから会話履歴をプレビュー
- **スパークアイコンボタン**（オレンジ角丸背景）でカードから Claude Code でセッションを直接開く

### ソート・グループ化

- **ソート** — 日付（新/古）・名前・メッセージ数・モデルで並び替え
- **グループ切替** — 日付別（デフォルト）/ タグ別 / エージェント別 / フラット（グループなし）
- ビュータイトルバーのボタンからワンクリックで切替

### セッション削除

- 不要なセッションを右クリック → 「セッションを削除」
- ファイルは `~/.claude/.trash/` に移動（完全削除はしない）
- 紐づき済みエージェントがある場合は紐づけ解除の確認あり
- ブックマーク・タグ・メモ・カスタム名も同時にクリーンアップ

### ウェルカム画面

- エージェント未登録時に「取締役を登録」ボタンを表示
- 取締役プリセット（Opus・固定・全体統括）で即座に開始可能

### メモリ管理

- **メモリ一覧** — タイプ別バッジ（user / feedback / project / reference）付きで表示
- **容量表示** — プログレスバー付きインデックス使用率（最大200行）
- **プレビュー・編集・統合・抽出・削除**
- **メモリパスをコピー** — 右クリックからファイルパスをコピー（他エージェントに読ませる際に便利）
- **OneDrive対応** — 日本語を含むOneDriveパスのプロジェクト名を正しく表示

## クイックスタート

### 1. インストール後

VS Code 左のアクティビティバーに 💬 アイコンが表示されます。クリックすると5つのセクションが表示されます:

- **会話一覧（セッション）** — すべてのClaude Code会話
- **会話ブックマーク** — お気に入り登録した会話
- **会話タグ** — タグ別に分類した会話
- **エージェント管理** — 登録済みエージェントの一覧
- **メモリ管理** — Claude Codeのメモリファイル

### 2. エージェントを登録する

1. 会話一覧からエージェント用のセッションを見つける
2. 右クリック → 「エージェントとして登録」
3. フォームに従って入力:
   - **部署名** — エージェントの識別名（例: CSM開発部）
   - **役割** — 担当業務（例: TypeScript開発・品質管理）
   - **モデル** — opus / sonnet / haiku から選択
   - **セッション運用** — 固定（同じセッションを使い続ける）/ 使い捨て（タスクごとに新セッション）
   - **親エージェント** — 階層構造がある場合に親を選択
   - **作業フォルダ** — エージェントの作業ディレクトリ
4. ルールファイル未指定の場合、ひな形の自動生成を提案されます

### 3. 組織図を確認する

ステータスバー左下の「📡 稼働中」表示をクリックすると、登録済みエージェントの組織図が開きます。

### 4. エージェントを管理する

- **エージェント管理サイドバー** で一覧を確認
- 右クリックで設定編集・セッション紐づけ変更・削除

## エージェント運用ガイド

### 1. 取締役の登録

マルチエージェント運用の第一歩は、全体を統括する「取締役」エージェントを登録することです。

- エージェント管理のウェルカム画面から「取締役を登録」をクリック
- **モデル**: Opus（高度な判断・指示出しが必要なため、最上位モデルを推奨）
- **セッション運用**: 固定（長期的な文脈を保持するため）
- **役割**: 全体統括・タスクの分割・指示出し・承認判断
- 取締役はルールファイルで「行動規範」「禁止事項」「判断基準」を定義しておくと効果的

### 2. 部署（子エージェント）の追加

取締役の下に、業務内容に応じた部署（子エージェント）を追加します。

- 会話一覧から対象セッションを右クリック → 「エージェントとして登録」
- **親エージェント**: 「取締役」を設定（組織図でツリー表示される）
- **モデルの使い分け**:
  - 重要な判断・設計タスク → Opus
  - 通常の開発・実装タスク → Sonnet
  - 定型作業・軽量タスク → Haiku（コスト効率が高い）

| 部署名 | モデル | 役割 |
|--------|--------|------|
| CSM開発部 | Opus / Sonnet | 機能実装・コードレビュー |
| テスト部 | Sonnet | テスト設計・自動化 |
| 調査部 | Sonnet | 技術調査・情報収集 |
| 雑務部 | Haiku | ドキュメント生成・定型作業 |

### 3. ルールファイルの育成

各エージェントにルールファイル（.md）を設定し、役割・行動規範・ノウハウを蓄積します。

- エージェントを右クリック → 「ルールファイルを編集」で直接編集
- 初期はひな形を自動生成し、運用しながら育てる
- フィードバックやノウハウを追記して成長させる
- ルールファイルはセッションをまたいで引き継がれる **長期記憶** として機能する

### 4. セッションの引き継ぎ

コンテキストウィンドウの上限に達したらセッションを更新します。

- エージェントを右クリック → 「セッションを新しくする」
- **引き継ぎメッセージ（遺言）** を入力し保存
- 旧セッションの紐づけが解除され、新セッションと再紐づけする

### 5. おすすめ構成例

**取締役 → 部署 → 班** の三階層構造:

```
取締役（Opus・固定）— 全体統括・タスク分割・最終判断
  ├── 開発部（Opus・固定）— 実装・レビュー・設計
  │     ├── 実装班A（Sonnet・固定）
  │     └── 実装班B（Sonnet・固定）
  ├── テスト部（Sonnet・使い捨て）— テスト設計・自動化
  ├── 調査部（Sonnet・使い捨て）— 技術調査・情報収集
  └── 雑務部（Haiku・使い捨て）— ドキュメント・定型作業
```
