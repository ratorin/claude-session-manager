エージェント設定項目

名前・役割

例: 「CSM開発部」「テスト担当」。組織図やサイドバーに表示される。

11エージェント運用ガイド

CSMを使ったマルチエージェント運用の実践的な手順を紹介します。取締役 → 部署 → 班の三階層構造で、大規模プロジェクトも効率よく管理できます。

ステップ1: 取締役を登録する

取締役の役割

全体統括・タスク分割・最終判断を担う最上位エージェント。ユーザーからの指示を受け取り、各部署に割り振る。

推奨設定

モデル: Opus（高度な判断に対応）
セッション運用: 固定（長期の文脈を保持）

エージェント管理

名前・モデル・
ルールを入力
→
セッションを
紐づけて
運用開始

ステップ2: 部署（子エージェント）を追加する

業務内容に応じて部署を追加します。親エージェントに「取締役」を設定することで、組織図にツリー表示されます。

モデルの使い分け: 重要な判断・設計 → Opus / 通常の開発・調査 → Sonnet / 定型・軽量タスク → Haiku（コスト効率◎）
固定 vs 使い捨て: 会話文脈を積み上げたい部署（開発部など）は「固定」、毎回クリーンな状態で動かしたい部署（調査部・テスト部など）は「使い捨て」がおすすめです。

ステップ3: ルールファイルを育てる

各エージェントにルールファイル（.md）を設定し、役割・禁止事項・ノウハウを蓄積します。

ルールに書くこと

  • エージェントの役割と担当範囲
  • 禁止事項（例: rm禁止 → .trash/に移動）
  • 使用ツールと作業フォルダ
  • 出力形式・言語の指定

育成のコツ

  • 作業中のフィードバックをその都度追記
  • 「毎回同じ指示」をルール化して省力化
  • プロジェクト固有のノウハウを蓄積
編集方法: エージェントを右クリック → 「ルールファイルを編集」でエディタが開きます。ルールファイルはセッションをまたいで引き継がれる長期記憶として機能します。
